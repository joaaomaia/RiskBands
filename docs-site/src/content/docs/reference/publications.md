## Status atual

Esta seção ainda não é um catálogo formal de papers, mas já deixou de ser
apenas um placeholder.

Hoje ela funciona como um mapa do que já está suficientemente maduro para ser
tratado como material público do projeto e do que ainda precisa evoluir antes
de virar publicação mais formal.

## O que já pode ser tratado como material público

O melhor material metodológico do RiskBands ainda vive no próprio repositório e
na documentação oficial:

- [Benchmark PD vintage](../../methodology/pd-vintage-benchmark/)
- [Por que RiskBands](../../methodology/why-riskbands/)
- [Como ler os gráficos](../../methodology/how-to-read-the-charts/)

Além dessas páginas, os artefatos públicos mais úteis hoje são:

- o notebook e o script do benchmark PD vintage
- os exemplos de champion challenger
- o README principal do repositório
- os visuais integrados à documentação oficial

## O que esta seção deve concentrar no futuro

Quando o projeto amadurecer mais em divulgação externa, esta página deve reunir:

- notas técnicas publicáveis
- write-ups de benchmark em formato mais editorial
- artigos curtos sobre robustez temporal em crédito
- apresentações, talks ou estudos de caso públicos

## Como usar esta página hoje

Se você chegou por aqui buscando a tese metodológica do projeto, o caminho mais
útil ainda é:

1. começar em [Por que RiskBands](../../methodology/why-riskbands/)
2. seguir para [Por que não usar apenas OptimalBinning](../../methodology/why-not-only-optimal-binning/)
3. terminar no [Benchmark PD vintage](../../methodology/pd-vintage-benchmark/)

## Limite honesto

Ainda não há paper oficial nem seção de publicações formais consolidada.

Nesta fase, esta página existe para organizar o que já é público, útil e
citável dentro do ecossistema atual do projeto, sem prometer uma produção
editorial que o repositório ainda não entrega.
