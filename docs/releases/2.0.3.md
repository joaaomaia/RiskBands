# RiskBands 2.0.3

Type: patch release

Status: local release candidate; not published yet.

## Highlights

- Improves categorical robustness for rare categories, missing values, and unknown categories with deterministic behaviour.
- Hardens `export_bundle(...)` against path traversal by sanitizing generated names and keeping a traceable manifest.
- Adds `force_numeric` for explicit numeric coercion control.
- Strengthens local and CI quality gates with `ruff`, `pytest-cov`, `pip check`, `bandit`, and `pip-audit`.
- Constrains the `optbinning`, `ortools`, and `protobuf` supply chain to avoid the vulnerable `ortools 9.11.4210 -> protobuf 5.26.1` resolver path.
- Updates README guidance for pandas support, current PySpark limitations, Spark/Databricks usage, overrides, auditable exports, assets, and prompt governance.
